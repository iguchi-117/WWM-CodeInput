# WWM コード高速入力ツール

ゲーム内コード入力ページ（ブラウザ等）に、WWM（Where Winds Meet / 燕雲十六声）の引き換えコードを**爆速で連続入力**するためのデスクトップアプリ。

## ✨ 特徴

- ✅ **ホットキー操作** — マウスに触らず連続でコード入力
- ✅ **JSON外部管理** — コードは `codes.json` で一元管理。エディタで直接編集OK
- ✅ **新規コード即対応** — アプリ起動中に `codes.json` を編集 → 5秒以内に自動再読込
- ✅ **使用済自動マーク** — 一度入力したコードはグレーアウト、二度と出ない
- ✅ **追加インストール不要** — 配布exe版は Python 不要

## 🚀 起動方法（3通り）

### 方法 A: 配布exe版（推奨・簡単）

`dist/WWMCodeInput.exe` をダブルクリック。
**Python インストール不要**、配布も簡単。

```bash
# exe を作るには
python build_exe.py
# → dist/WWMCodeInput.exe が生成される
```

配布: `WWMCodeInput.exe` を zip にして送るだけ。受け取った側はダブルクリックで起動。

### 方法 B: start.bat ダブルクリック

Python 3.11+ がインストールされていれば、`start.bat` をダブルクリックで起動。
exe ビルド不要、ソース変更が即反映。

```bash
# 配布先に Python がある場合
start.bat
```

### 方法 C: コマンドライン

```bash
python app.py
```

## ⌨️ ホットキー一覧

### アプリ内（フォーカスがウィンドウにある時）
| キー | 動作 |
|------|------|
| `Ctrl+C` | 選択行のコードをクリップボードにコピー |
| `Alt+G` | **次の未使用コード**を自動コピー（メイン操作） |
| `Ctrl+Shift+U` | 選択中のコードを全部「使用済」にマーク |
| `Delete` | 選択行を使用済にマーク |
| `Insert` | 新規コード一括追加ダイアログ |
| `F5` | codes.json を再読込 |
| `Ctrl+,` | 設定ダイアログ（ホットキー変更など） |

### グローバル（ゲームにフォーカスがあっても効く）
| キー | 動作 |
|------|------|
| `Alt+G` | **次の未使用コード**をクリップボードにコピー（デフォルト） |

> 💡 **推奨ワークフロー**:
> 1. アプリ起動 → `Alt+G` で1件目をコピー
> 2. ゲームにフォーカス → コード入力欄に `Ctrl+V`
> 3. **入力成功** → ゲームからアプリに戻らず `Alt+G` で次のコードがコピーされる
> 4. そのままゲームにフォーカス → `Ctrl+V` で2件目…
>
> このサイクルなら**5秒/1コード**で連続入力できます。

## ⚠ ゲーム中 (DirectX フルスクリーン) で動作させるには

**管理者権限で起動** する必要があります。一部のアンチチートソフトや DirectX の
フルスクリーン排他モードは、`pynput` の `WH_KEYBOARD_LL` キーボードフックを
ブロックするためです。

### 設定手順

1. **デスクトップの「WWM Code Input」ショートカットを右クリック**
2. **「プロパティ」** を開く
3. **「ショートカット」タブ** → **「詳細設定(V)...」**
4. **「管理者として実行(R)」** にチェック → **OK**
5. **「適用(A)」** → **「OK」**

これでショートカットをダブルクリックすると UAC ダイアログが出ます。
「はい」を押せば管理者権限で起動し、ゲーム中でもホットキーが効きます。

> 💡 設定後はショートカットに **盾マーク (管理者アイコン)** がつきます。

      "used_at": null,
      "source": "Discord (2026-07-02 ユーザー報告)"
    }
  ]

### 新規コードを追加する方法（3通り）

#### A. アプリ内で追加（推奨）
1. アプリ起動中に `Insert` キー（または「＋ 追加」ボタン）
2. ダイアログにコード入力 → Enter
3. 自動的に `codes.json` に追記 + UI に反映

#### B. ファイルを直接編集
`codes.json` の `codes` 配列末尾に新しいオブジェクトを追加：

```json
{
  "code": "NEWCODE123",
  "used": false,
  "added_at": "2026-07-02",
  "used_at": null,
  "source": "手動追加"
}
```

アプリを起動していれば 5秒以内に自動反映。起動していなければ次に起動した時。

#### C. スクリプトから追加
```python
import json
with open("codes.json", encoding="utf-8") as f:
    data = json.load(f)
data["codes"].append({
    "code": "NEWCODE123", "used": False,
    "added_at": "2026-07-02", "used_at": None, "source": "スクリプト"
})
with open("codes.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
```

## 🛠 機能詳細

### ペースト後の自動「次へ」コピーについて

**技術的制約**: `Ctrl+V` はクリップボードを変更しない（読むだけ）ため、「ペーストした瞬間」を純粋に検知するのは不可能です。

**本アプリの解決策**:
1. **メイン方式**: グローバルホットキー `Ctrl+Shift+G`（ゲームにフォーカス中でも効く）
   - 「ペースト終わった」→「`Ctrl+Shift+G`押す」→ 自動で使用済マーク + 次をコピー
2. **補助方式**: アプリにフォーカスが戻ったとき（Alt+Tab）にステータスバーで「次へコピーできます」と通知
3. **手動方式**: アプリに切り替えて `Ctrl+G` で次をコピー

これにより「**5秒で1コード**」の高速入力サイクルを実現しています。

### アトミック書き込み
- `codes.json.tmp` に書いてから `os.replace()` でリネーム
- 書き込み中にクラッシュしても既存ファイルは無傷
- 毎回 `.backup.json` も自動生成

### 重複防止
- 同じコード（大文字小文字区別なし）が既にあれば追加ダイアログで警告
- `codes.json` を直接編集した場合も、起動時にチェック

### 外部ファイル変更の自動検知
- 5秒ごとに `codes.json` の mtime を確認
- 変更があれば自動再読込 → Treeview 更新

### exe 化での `codes.json` 取り扱い
- PyInstaller で `--onefile` ビルド時、`codes.json` は exe 内部に同梱
- exe 起動時に exe と同じフォルダに展開（**`app.py` で `sys.frozen` を検出して自動対応**）
- 配布exe初回起動時、`codes.json` が存在しなければ exe と同じフォルダに空のものを自動生成
- 既存 `codes.json` を編集する場合は exe と同じフォルダの `codes.json` を開く

## 🐛 トラブルシューティング

### 「⚠ グローバルホットキー Ctrl+G の登録に失敗しました」と表示される
- 他のアプリが `Ctrl+G` をグローバルホットキーとして使っている可能性
- 代替: アプリ内で `Ctrl+G` を使う（フォーカスをアプリに戻して操作）

### クリップボードへのコピーが動かない
- 一部リモートデスクトップ環境では `OpenClipboard` が失敗する場合あり
- その場合は「＋ 追加」ダイアログでコードを追加後、ダブルクリックで再試行

### exe版で codes.json が読み込まれない
- セキュリティソフトが `_MEIPASS` 展開をブロックしている可能性
- exe と同じフォルダに手動で `codes.json` を配置してください

### Windows Defender が exe をブロック
- 「詳細情報」→「実行」で起動可能
- 配布時は VirusTotal 等でチェック推奨

### アプリが起動しない（Python版）
- Python 3.11+ をインストール: https://www.python.org/downloads/
- インストール時に **"Add Python to PATH"** にチェック
- コマンドプロンプトで `python --version` で確認
